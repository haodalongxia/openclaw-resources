# 飞书开放平台 - 事件配置指南

## 接收消息 2.0 是什么？

「接收消息 2.0」是飞书机器人的消息接收方式，需要配置后才能让机器人收到群聊/私聊消息。

---

## 配置步骤

### 步骤 1: 进入应用管理

1. 访问 https://open.larksuite.com/app/cli_a93e6722b4615bd8
2. 登录飞书开放平台
3. 进入「king」应用管理后台

### 步骤 2: 找到事件配置

1. 左侧菜单 → 「事件与回调」
2. 找到「事件配置」部分
3. 点击「添加事件」或「编辑」

### 步骤 3: 配置接收消息 2.0

**需要勾选的事件**:

| 事件 | 说明 | 必须 |
|------|------|------|
| `im.message.receive_v1` | 接收消息 2.0 | ✅ 必须 |
| `im.message.read_v1` | 消息已读 | ⭕ 可选 |
| `chat.update` | 群聊信息更新 | ⭕ 可选 |

**配置详情**:

```
事件类型：im.message.receive_v1
接收方式：HTTP URL
URL: https://你的服务器地址/webhook/feishu
Token: 自动生成（保存好）
Encrypt Key: 自动生成（保存好）
```

### 步骤 4: 权限配置

在「权限管理」中添加：

| 权限 | 说明 |
|------|------|
| 获取与发送单聊、群组消息 | 接收和发送消息 |
| 获取用户信息 | 获取发送者信息 |
| 获取群组信息 | 获取群聊信息 |

### 步骤 5: 发布应用

1. 点击右上角「版本管理与发布」
2. 点击「创建版本」
3. 填写版本号（如 v1.0.0）
4. 点击「发布」
5. 等待审核（通常几分钟）

如果没有公网 IP，使用 ngrok:


## 常见问题

**Q: 事件配置后收不到消息？**
A: 检查 URL 是否可访问，Token 是否正确

**Q: 提示「URL 验证失败」？**
A: 确保 feishu-bridge 已启动，URL 能访问

**Q: 群聊里@机器人没反应？**
A: 检查是否添加了「接收群消息」权限

**Q: 需要备案域名吗？**
A: 国内服务器需要，用 ngrok 临时测试不需要
